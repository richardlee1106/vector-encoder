## 二、核心发现

### 1. 数据特性决定理论上限

**发现**：武汉POI空间分布均匀，KMeans聚类理论上限稳定在0.32-0.38。

| 数据类型 | 理论上限(Sil) | 原因 |
|----------|---------------|------|
| 随机采样 | 0.32-0.38 | 包含多个聚类中心 |
| 单区域 | -0.3~-0.4 | POI分布过于均匀 |

**教训**：评估模型前先计算理论上限。

### 2. 高维映射的维度诅咒

**现象**：Pearson=0.97（整体距离关系正确），但邻居重叠率只有~30%。

**原因**：真实空间2维，Embedding空间64维。

**解决**：邻域一致性损失 + KNN特征。

### 3. 稳定性 > 绝对值

| 版本 | 达成率 | 稳定性 |
|------|--------|--------|
| V2.3 | 12%-91% | ❌ 波动大 |
| V2.4 | 86%-89% | ✅ 稳定 |

**教训**：固定随机种子 + 优化训练策略。

### 4. 标签泄露问题（V6/V61）

**问题**：category同时作为输入和标签，导致Silhouette虚高（0.98+）。

**教训**：必须使用与输入无关的标签（如空间聚类）验证真实学习能力。

## 三、实验方法论

### 渐进式验证流程

```
Phase 1: 单样本验证 (10%采样)
         ↓ 通过: 达成率>70%, Pearson>0.7, 重叠率>20%
Phase 2: 三比例验证 (5%/10%/20%)
         ↓ 通过: 3/3比例通过
Phase 3: 渐进式全量 (10%→30%→60%→80%→100%)
         ↓ 每阶段检查性能下降
```

### 实验心得

| 心得 | 发现 | 对策 |
|------|------|------|
| 数据特性定上限 | 单区域理论上限为负 | 用采样数据验证 |
| 维度诅咒 | Pearson高但重叠率低 | 邻域一致性损失 |
| 稳定性优先 | 波动大=不可靠 | 固定种子+优化 |
| 目标对齐 | 训练vs评估不一致 | 多任务损失 |
| 区域选择 | 特定区域不适合验证 | 使用随机采样 |

## 七、V2.6 设计原则

### 核心目标

构建**统一空间基础层**，为上层应用（RAG、LLM推理、空间分析）提供标准化接口。

### 关键约束

1. **性能**：支持84万POI + 10万线/面
2. **扩展性**：支持增量更新
3. **可解释性**：embedding具备空间语义
4. **标准化**：LLM/RAG友好接口

### 技术选型

| 组件 | V2.4方案 | V2.6方案 |
|------|----------|----------|
| 坐标系统 | 经纬度直接编码 | 极坐标 + 多尺度编码 |
| 邻域特征 | KNN固定K | 距离加权邻域 |
| 损失函数 | 固定权重 | 自适应权重 |
| 稀疏处理 | 无 | ZIP分布 |
| 功能区 | 无 | 对比学习 |
