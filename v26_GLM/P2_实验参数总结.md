### 核心问题诊断

**Region Sep = 类间距离 / 类内距离 = 0.65**

**问题根源**：
1. embedding 层专注于距离保持（Pearson=0.9784），缺乏语义结构
2. 分类能力被剥离到 hidden 层的分类头，绕过了 embedding 层
3. 类间距离 < 类内距离，不同功能区在特征空间中重叠严重

## 二、P2 优化方案

### 核心策略：Center Loss

**原理**: 让同类 embedding 向可学习的类别中心聚拢，在保持距离损失的同时引入温和的语义约束。

**优势**：
- ✅ 只拉近类内距离，不直接推开类间距离
- ✅ 与距离损失兼容性好
- ✅ 权重可调，避免破坏 Pearson
- ✅ 实现简单，训练开销小

### Center Loss 实现

```python
class CenterLoss(nn.Module):
    """
    类别中心损失 - 让同类 embedding 向类中心聚拢
    
    L_center = Σ ||embedding_i - center_{y_i}||^2
    
    参考: A Discriminative Feature Learning Approach for Deep Face Recognition (ECCV 2016)
    """
    
    def __init__(self, num_classes=6, feat_dim=352, alpha=0.5):
        super().__init__()
        self.num_classes = num_classes
        self.feat_dim = feat_dim
        self.alpha = alpha  # 中心更新速率
        
        # 可学习的类别中心 [num_classes, feat_dim]
        self.register_buffer('centers', torch.randn(num_classes, feat_dim))
    
    def forward(self, embeddings, labels):
        # 过滤未知标签（label=6）
        valid_mask = labels < 6
        if valid_mask.sum() == 0:
            return torch.tensor(0.0, device=embeddings.device, requires_grad=True)
        
        embeddings = embeddings[valid_mask]
        labels = labels[valid_mask]
        
        # 归一化中心向量（与 L2 归一化的 embedding 保持一致）
        centers_norm = F.normalize(self.centers, p=2, dim=1)
        centers_batch = centers_norm[labels]
        
        # 计算损失
        loss = torch.pow(embeddings - centers_batch, 2).sum(dim=1).mean()
        
        # 动量更新中心
        with torch.no_grad():
            for label in labels.unique():
                mask = labels == label
                if mask.sum() > 0:
                    delta = (embeddings[mask] - centers_norm[label]).mean(dim=0)
                    self.centers[label] += self.alpha * delta
        
        return loss
```

### 损失函数集成

```python
# 总损失函数
total_loss = (
    distance_weight * l_distance +           # 0.5 - 距离保持
    reconstruction_weight * l_reconstruct +  # 0.3 - 坐标重建
    direction_weight * l_direction +         # 1.5 - 方向识别
    region_weight * l_region +               # 1.5 - 功能区分类
    center_weight * l_center                 # 0.2 - 类别中心（新增）
)
```

## 七、风险评估与缓解

### 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Pearson 下降过多（< 0.95） | 中 | 高 | 使用小权重（0.1-0.2），逐步调优 |
| Region Sep 提升不足（< 1.5） | 低 | 中 | 进入 Phase 3，添加 Triplet Loss |
| 训练时间大幅增加（> 20%） | 低 | 低 | Center Loss 计算开销小，可接受 |
| 过拟合（验证集性能下降） | 中 | 中 | 监控验证集，使用 Dropout=0.15 |
| 类别中心不收敛 | 低 | 中 | 调整 alpha（中心更新速率） |

### 回滚策略

如果 P2 实验导致性能严重下降：
1. 回退到 P1F-Final 配置（center_weight=0.0）
2. 尝试更小的 center_weight（0.05-0.1）
3. 考虑替代方案（如只在 hidden 层应用 Center Loss）
